# Weekly Reflections

用途：作为 L2 周反思的持久化日志，记录从 `OBSERVATIONS.md` 提炼出的重复模式、偏好候选和晋升判断。  
定位：这是连接 L1 原始观察与 L2.5 / L3 长期结构的桥梁，不替代 `PREFERENCE_REGISTRY.md`，而是为它提供证据链。

## 使用原则

- 采用 append-only 模式，每周追加一个 `## Week Of: YYYY-MM-DD` 条目
- 每个结论都尽量回指具体 observation 日期、项目或文件
- 优先写稳定信号和边界条件，不要把一周流水重新抄一遍
- 即使本周没有产生新偏好，也要记录为什么没有晋升
- 一条偏好若进入 `PREFERENCE_REGISTRY.md`，尽量在反思中留下对应的 `PREF-ID`

## 最小结构

每周条目至少覆盖：

1. 输入窗口与覆盖项目
2. 重复出现的模式
3. 审稿 / 识别 / 写作 / 协作中的高信号判断
4. 新增或更新的长期偏好
5. 需要晋升或降级的内容
6. 下周 keep / stop / start

## 与其他文件的关系

- `OBSERVATIONS.md`：保存原始痕迹
- `reflector_weekly_template.md`：提供本文件的写作模板
- `PREFERENCE_REGISTRY.md`：保存已经稳定但仍可调整的偏好
- `MEMORY_PROMOTION_PROTOCOL.md`：决定何时升到 `rules/`、`axioms/` 或 `skills/`
